# YouTube Script — AI Music Lab

Строй чёткий путь от обещания зрителю до доказанного результата. Оптимизируй под понимание и удовлетворение, не под искусственную задержку.

## Контекст канала

- **Канал:** AI Music Lab — туториалы по Suno AI и авторские треки
- **Язык:** русский
- **Аудитория:** русскоязычные создатели контента и музыканты, интересующиеся AI музыкой
- **Long-form:** 5–12 минут, туториал / объяснение / эксперимент / сравнение
- **Shorts:** 30–60 секунд, один приём или один результат
- **Голос:** конкретный, без воды, с живой деталью; без слов AI / нейросеть в заголовке
- **Финал Long:** переход к собственному треку + направление в плейлист

## Контракт перед написанием

Зафиксируй:
- кто зритель и что он уже знает
- формат (Long / Shorts) и желаемая длина
- обещание заголовка и миниатюры
- какое доказательство есть (демо, до/после, A/B, скрин)
- голос канала: прямой, разговорный, без саморекламы в начале

## Выбор структуры

Читай `references/script-frameworks.md`. Выбирай один фреймворк под задачу зрителя:

| Задача зрителя | Long-form | Shorts |
|---|---|---|
| Повторить действия | L01 reproducible tutorial | S01 result-first micro-tutorial |
| Понять механизм | L02 guiding question, L07 progressive depth | S02 question/reveal |
| Исправить ошибку | L03 misconception, L05 diagnosis tree | S03 mistake/fix |
| Сравнить варианты | L06 transparent experiment | S04 A/B criterion |
| Показать кейс | L04 evidence-aware case study | S06 micro-story |
| Визуальная трансформация | L04 case study | S05 visual turn |

Не загоняй всё в одну формулу — выбирай под реальный контент.

## Написание по битам

1. Подтверди клик хуком выровненным под упаковку
2. Дай минимальный контекст для понимания результата
3. Веди прогресс через вопросы, шаги, тесты или нарастающие доказательства
4. Размещай ре-хуки на реальных переходах или незакрытых вопросах
5. Доставь обещанный результат явно
6. Заверши коротким выводом или релевантным следующим шагом

**Для туториалов:** сохраняй предпосылки и причинно-следственные шаги даже при сокращении.
**Для Shorts:** один тезис — один визуальный поворот — один результат; без сжатого длинного формата.

**Финал Long-form (обязательный блок):**
```
«Кстати, вот трек который я сделал используя именно этот приём —
послушай первые 30 секунд...»
[трек — 30–45 секунд]
«Полный трек — в плейлисте [название], ссылка в описании.»
```

## Проверка каждого бита

Пометь каждый бит: `обещание` / `контекст` / `доказательство` / `объяснение` / `прогресс` / `результат` / `CTA`.
Убирай или объединяй биты без функции для зрителя. Флагируй: неподтверждённые утверждения, скрытые предпосылки, повторы, преждевременные CTA, разделы где визуал дублирует текст.

## Размещение CTA

- Сначала главный обучающий результат — потом CTA
- Предпочитай релевантное следующее действие: тест, чеклист, следующий урок, файл настроек
- Mid-roll CTA — только на естественной границе раздела
- Никогда не прерывай незакрытый технический шаг

## Вывод

Возвращай:

1. аудитория и обещание
2. выбранный фреймворк и причина выбора
3. битовый лист с таймингом или полный сценарий (по запросу)
4. заметки по визуалам и доказательствам
5. риски удержания и проверка утверждений
6. варианты (помечены как гипотезы)

Для работы только с заголовком/миниатюрой — используй `youtube-packaging`.
Для первых 5–30 секунд отдельно — используй `youtube-hooks`.
